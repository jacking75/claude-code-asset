# ✨ CSharpier 사용 방법 완벽 정리

**CSharpier란?**

CSharpier는 C# 및 XML을 위한 **독선적인(opinionated) 코드 포매터**입니다. [Prettier](https://prettier.io/)의 철학에서 영감을 받아, 코드를 파싱하고 자체 규칙에 따라 재출력합니다. 설정 옵션을 최소화하여 팀 전체가 동일한 코드 스타일을 유지할 수 있도록 도와줍니다.

---

## **1. 설치 (Installation)**
CSharpier는 .NET 도구(dotnet tool)로 제공되며, **로컬(Local)** 또는 **전역(Global)** 방식으로 설치할 수 있습니다.

**로컬 설치 (권장)**

팀 전체가 동일한 버전을 사용하도록 프로젝트별로 설치하는 방식입니다.

```bash
# tool-manifest 파일이 없는 경우 먼저 생성
dotnet new tool-manifest

# CSharpier 로컬 설치
dotnet tool install csharpier
```

**전역 설치**

```bash
dotnet tool install -g csharpier
```

**업데이트**

```bash
# 로컬 업데이트
dotnet tool update csharpier

# 전역 업데이트
dotnet tool update csharpier -g
```

## **2. CLI 사용법 (Command Line Interface)**

**기본 포맷 명령어**

현재 디렉토리와 하위 디렉토리의 모든 C# 파일을 포맷합니다.

```bash
dotnet csharpier format .
```

특정 파일이나 디렉토리를 지정할 수도 있습니다.

```bash
# 특정 파일 포맷
dotnet csharpier format ./src/MyClass.cs

# 특정 디렉토리 포맷
dotnet csharpier format ./src
```

**주요 Format 옵션**

| 옵션 | 설명 |
|---|---|
| `--config-path` | 설정 파일 경로를 직접 지정 |
| `--ignore-path` | 무시 파일 경로를 직접 지정 |
| `--log-level` | 로그 레벨 설정 (`None`, `Error`, `Warning`, `Information`, `Debug`) |
| `--log-format` | 로그 출력 형식 (`Console`, `MsBuild`) |
| `--no-cache` | 캐시를 무시하고 강제 포맷 |
| `--write-stdout` | 파일을 수정하지 않고 결과를 stdout으로 출력 |
| `--include-generated` | 자동 생성 파일도 포맷 대상에 포함 |
| `--skip-validation` | 포맷 후 구문 트리 검증 생략 |
| `--skip-write` | 실제 파일 쓰기 없이 검사만 수행 |
| `--compilation-errors-as-warnings` | 컴파일 오류를 경고로 처리 |

```bash
# 설정 파일 경로를 직접 지정하는 예시
dotnet csharpier format . --config-path "./config/.csharpierrc"

# 로그 레벨을 Debug로 설정
dotnet csharpier format . --log-level Debug
```

**Check 명령어 (CI/CD에 유용)**

파일이 이미 포맷되어 있는지 확인만 하고, 수정되지 않은 파일이 있으면 종료 코드 1을 반환합니다.

```bash
dotnet csharpier check .

# 미포맷 파일을 오류가 아닌 경고로 처리
dotnet csharpier check . --unformatted-as-warnings
```

**Server 모드**

IDE 플러그인 등에서 활용하는 HTTP 서버 모드입니다. 단일 파일 포맷 시 dotnet 시작 오버헤드를 줄이기 위해 사용합니다.

```bash
dotnet csharpier server

# 특정 포트 지정
dotnet csharpier server --server-port 8082
```

**stdin 파이프 입력**

```bash
# 파이프로 코드 입력 후 stdout으로 결과 출력
"public class Class { }" | dotnet csharpier format --stdin-path "MyFile.cs"

# cat으로 파일을 읽어 포맷 결과 확인
cat TestFile.cs | dotnet csharpier
```

## **3. 설정 파일 (Configuration)**

CSharpier는 아래 형식의 설정 파일을 지원합니다. 파일을 포맷할 때 해당 파일 위치에서 상위 디렉토리를 순서대로 탐색하여 설정을 적용합니다.

- `.csharpierrc` (JSON 또는 YAML)
- `.csharpierrc.json` / `.csharpierrc.yaml`
- `.editorconfig`

**설정 옵션 (JSON 예시)**

```json
{
  "printWidth": 100,
  "useTabs": false,
  "indentSize": 4,
  "endOfLine": "auto"
}
```

**설정 옵션 (YAML 예시)**

```yaml
printWidth: 100
useTabs: false
indentSize: 4
endOfLine: auto
```

**각 옵션 설명**

- **`printWidth`** — 줄바꿈이 발생하는 기준 너비입니다. 엄격한 제한이 아니므로 일부 줄은 이보다 길거나 짧을 수 있습니다. 기본값은 `100`입니다.
- **`useTabs`** — 들여쓰기에 스페이스 대신 탭 문자를 사용할지 결정합니다. 기본값은 `false`입니다.
- **`indentSize`** — 들여쓰기 한 단계당 스페이스 수를 지정합니다. C#의 기본값은 `4`, XML은 `2`입니다.
- **`endOfLine`** — 줄 끝 문자를 지정합니다. `auto` (기존 유지), `lf` (Linux/macOS), `crlf` (Windows) 중 선택할 수 있으며, 기본값은 `auto`입니다.

**파일별 설정 오버라이드 (Overrides)**

glob 패턴을 사용해 특정 파일 확장자에 다른 설정을 적용할 수 있습니다.

```json
{
  "overrides": [
    {
      "files": "*.{csr,cst}",
      "formatter": "csharp",
      "indentSize": 2,
      "useTabs": true,
      "printWidth": 10,
      "endOfLine": "LF"
    }
  ]
}
```

**EditorConfig 사용 예시**

```ini
[*.{cs,csx}]
charset = utf-8
insert_final_newline = true
trim_trailing_whitespace = true
indent_style = space
indent_size = 4
max_line_length = 100

[*.{config,csproj,props,slnx,targets,xaml,xml}]
indent_style = space
indent_size = 2
max_line_length = 100
```

## **4. 파일 무시 (Ignore)**

**`.csharpierignore` 파일**

`.gitignore`와 동일한 문법을 사용하여 포맷 대상에서 제외할 파일/폴더를 지정합니다.

```
# .csharpierignore
Uploads/**/
App_Data/*.cs
```

CSharpier는 `.gitignore`도 자동으로 읽으며, `.gitignore`에서 무시된 파일을 포맷하고 싶다면 `.csharpierignore`에서 부정 패턴(`!`)으로 재포함할 수 있습니다.

```
# .csharpierignore
!**/generated
```

**기본적으로 무시되는 파일** — `TemporaryGeneratedFile_`로 시작하는 파일, `.designer.cs`, `.generated.cs`, `.g.cs`, `.g.i.cs`로 끝나는 파일, `<autogenerated` 또는 `<auto-generated` 주석으로 시작하는 파일, 그리고 `node_modules`, `obj`, `.git` 디렉토리 등이 기본적으로 제외됩니다.

**코드 내 특정 구간 무시**

```csharp
// 다음 노드 하나만 무시
// csharpier-ignore
public class Unformatted     { }

// 범위 무시
// csharpier-ignore-start
public class Unformatted1     { }
public class Unformatted2     { }
// csharpier-ignore-end

// 무시 이유 설명 추가 (v0.23.0+)
// csharpier-ignore - 외부 프로젝트에서 그대로 복사된 클래스
public class Unformatted     { }
```

## **5. 에디터 연동 (Editors)**

저장 시 자동 포맷(Format on Save)을 지원합니다. 공식 플러그인이 아래 에디터에서 제공됩니다.

- **Visual Studio** — Manage Extensions 대화상자에서 "CSharpier" 검색 후 설치
- **Visual Studio Code** — Extensions 사이드바에서 "CSharpier - Code Formatter" 검색 후 설치
- **Rider / JetBrains IDE** — Plugins 대화상자에서 "CSharpier" 검색 후 설치
- **Zed** — 아래와 같이 `settings.json`에 수동 설정

```json
{
  "languages": {
    "CSharp": {
      "format_on_save": "on",
      "formatter": {
        "external": {
          "command": "dotnet",
          "arguments": ["csharpier", "format"]
        }
      }
    }
  }
}
```

## **6. Pre-commit 훅 연동**

[Husky](https://typicode.github.io/husky/)를 사용하여 커밋 전에 자동으로 포맷을 실행할 수 있습니다. `.husky/task-runner.json` 파일을 다음과 같이 구성합니다.

```json
{
  "tasks": [{
    "name": "Run csharpier",
    "command": "dotnet",
    "args": ["csharpier", "format", "${staged}"],
    "include": [
      "**/*.cs",
      "**/*.csx",
      "**/*.csproj",
      "**/*.props",
      "**/*.targets",
      "**/*.xml",
      "**/*.config"
    ]
  }]
}
```

CI/CD 파이프라인에서 Husky를 비활성화하려면 환경변수를 다음과 같이 설정합니다.

```bash
HUSKY=0
```

## **7. CI/CD 연동**

CI 파이프라인에서는 `check` 명령어를 사용하여 모든 파일이 포맷되어 있는지 검사합니다. 미포맷 파일이 있으면 종료 코드 1을 반환하여 빌드를 실패시킵니다.

```bash
dotnet csharpier check .
```

특정 버전을 고정하려면 `.config/dotnet-tools.json`에 버전을 명시합니다.

```json
{
  "version": 1,
  "isRoot": true,
  "tools": {
    "csharpier": {
      "version": "1.0.0",
      "commands": ["csharpier"]
    }
  }
}
```

## **8. 포맷 전후 비교 예시**

**Before (포맷 전)**

```csharp
public class ClassName {    public void CallMethod() {         var shuffle = shuffle.Skip(26).LogQuery("Bottom Half").InterleaveSequenceWith(shuffle.Take(26).LogQuery("Top Half"), shuffle.Skip(26).LogQuery("Bottom Half")).LogQuery("Shuffle").ToArray();    }}
```

**After (포맷 후)**

```csharp
public class ClassName
{
    public void CallMethod()
    {
        var shuffle = shuffle
            .Skip(26)
            .LogQuery("Bottom Half")
            .InterleaveSequenceWith(
                shuffle.Take(26).LogQuery("Top Half"),
                shuffle.Skip(26).LogQuery("Bottom Half")
            )
            .LogQuery("Shuffle")
            .ToArray();
    }
}
```
